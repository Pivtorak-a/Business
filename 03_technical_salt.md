# Technical Salt

## Known facts

Уже зафіксовано:

- технічна сіль є одним із ключових напрямів системи;
- вона входить у модель бізнесу на площадці **0,4 га**;
- по сезонній логіці: **навесні — купівля солі**, **взимку — продаж**;
- для солі передбачено **навіс**;
- у юридичній моделі цей блок входить у контур **ФОП 3 група**.

## Сенс напряму

Сіль — це напрям із сильним сезонним перекосом:

- гроші вкладаються раніше;
- реалізація відбувається пізніше;
- зберігання критично впливає на якість;
- наявність запасу в правильний момент може давати сильну перевагу.

## Базова логіка

1. Закупка в сезон нижчого попиту або вигіднішої ціни.
2. Завезення на площадку.
3. Правильне зберігання.
4. Поступова або пікова реалізація у зимовий сезон.
5. За потреби — продаж разом із доставкою.

## Стратегічна роль

Сіль дає системі:

- зимовий сезонний дохід;
- складський товар із потенціалом обігу великими партіями;
- можливість поєднувати товар + логістику;
- аргумент для B2B роботи;
- можливість працювати з конкурентами через товар або перевезення.

## Інфраструктурні вимоги

Already known:

- потрібен **навіс під сіль**;
- площадка повинна мати хоча б часткове якісне покриття у ключових зонах;
- логістика і техніка мають бути інтегровані в модель зберігання і відвантаження.

## Data gaps

Поки що не зафіксовано:

- формат закупки: насип / біг-беги / фасовка;
- закупівельна ціна по сезонах;
- цільовий обсяг запасу;
- втрати при зберіганні;
- потреба в оборотці;
- тип клієнтів;
- швидкість обігу;
- середній чек;
- мінімально вигідна партія;
- чи буде фасування.

## Що треба буде рахувати

- landed cost на тонну;
- storage cost per ton per month;
- working capital lock period;
- winter sales curve;
- transport cost per delivery;
- spoilage / moisture loss;
- gross and net margin.

## Ключові ризики

- зайти в запас занадто рано або занадто дорого;
- не мати достатнього навісу;
- заморозити занадто багато оборотки;
- помилитись із масштабом зимового попиту;
- отримати низьку оборотність;
- переоцінити маржу без врахування доставки й перевалки.

## Роль у системному симуляторі

Сіль — це один із головних кейсів для моделювання:

- сезонної закупки;
- заморожування оборотного капіталу;
- складської інфраструктури;
- конфлікту за площу;
- логістики відвантаження;
- зимового cash flow.
